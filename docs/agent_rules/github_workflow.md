# Git / GitHub 運用ルール

## 基本

- Gitはセーブポイント。
- GitHubはクラウド側のバックアップ・履歴確認場所。
- AIに大きな変更をさせる前後は、可能ならcommitする。
- pushは作業の区切り、またはクラウドにも退避したい時に行う。

## commitとpushの違い

- commit: ローカルPC内にセーブする。
- push: commitした内容をGitHubへ送る。

## ユーザーが「プッシュして」と言ったとき

1. `git status` を確認する。
2. 変更がなければ終了する。
3. `git diff --stat` を確認する。
4. 必要ならCodex CLIに `git diff` の要約とcommitメッセージ案を作らせる。
5. `.env`、APIキー、トークン、個人情報、大容量データ、`external_repos/` が含まれていないか確認する。
6. 問題なければ `git add .`、`git commit -m "..."`、`git push` を行う。
7. 実行結果を報告する。

## 禁止事項

- ユーザーの許可なく `git reset --hard` を実行しない。
- ユーザーの許可なく `git restore .` を実行しない。
- ユーザーの許可なく `git push --force` を実行しない。
- 外部リポジトリ本体を自分のリポジトリへ混ぜない。

## commitメッセージの粒度

良い例：

- `点群読み込み処理を追加`
- `Codexレビューに基づき例外処理を追加`
- `論文調査メモと実装計画を追加`

悪い例：

- `fix`
- `test`
- `いろいろ修正`
